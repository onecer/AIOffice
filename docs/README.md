# AIOffice Documentation

**English** | [简体中文](README.zh-CN.md)

AIOffice is an AI-native Office engine: a single self-contained binary that lets
agents create, inspect, edit, render, validate, audit, diff, and convert real
`.docx`, `.xlsx`, and `.pptx` files. The important idea is not "another Office
library"; it is an execution surface with stable paths, one JSON envelope per
call, recoverable errors, and a visual loop that lets an agent see what it made.

## What It Does

AIOffice exposes the same document engine through a shell CLI and a stdio MCP
server. The current source tree has **19 CLI verbs** and **19 MCP tools**:
the document contract is still the stable `surfaceVersion: "1.0"`, while newer
preview tools and the CLI-only `plugin` installer improve the surrounding
workflow. When in doubt, ask the binary instead of guessing:

```bash
aioffice doctor
aioffice schema
aioffice help
aioffice help properties-pptx
```

The user-facing promise is deliberately simple:

- One command prints exactly one JSON envelope: `{ ok, data, error, meta }`.
- `error.suggestion` is always present; bad paths include `candidates`.
- Elements have stable canonical paths such as `/body/p[3]`, `/Sheet1/A1:C10`,
  and `/slide[2]/shape[@id=7]`.
- Edits are atomic batches and snapshot the pre-image before writing.
- `validate` checks file health; `audit` checks accessibility and quality.
- `render` and `preview` make visual review part of the agent loop.

## Three Formats, One Editing Model

Each Office format has format-native depth, but the agent does not need three
separate workflows. It reads, queries, gets, edits, validates, audits, renders,
diffs, and converts through the same command shape.

![Capability map for docx, xlsx, and pptx](assets/overview/format-map.svg)

### Word `.docx`

Use AIOffice for reports, memos, letters, contracts, and review documents.
Notable surfaces include headings, paragraph/run styles, tables, comments,
tracked changes, content controls, fields, footnotes/endnotes, citations,
bibliographies, equations, page setup, watermarks, and mail merge.

### Excel `.xlsx`

Use it for dashboards, budgets, data exports, and analysis workbooks. It can
write and read cells/ranges, styles, formulas with cached values, charts,
pivots, slicers, conditional formats, sparklines, data validation, scenarios,
goal seek, print setup, camera-tool snapshots, and CSV import/export.

### PowerPoint `.pptx`

Use it for decks, diagrams, launch slides, and briefings. It can position
shapes, images, tables, native charts, equations, SmartArt, connectors, groups,
masters/layouts, notes, sections, transitions, animations, embedded fonts,
action buttons, media, 3D model placeholders, and slide/section zoom links.

## The Visual Loop

Validation proves the OOXML is structurally sound. It does not prove the slide,
sheet, or page looks good. For any user-facing document, use a visual loop:

```bash
aioffice render deck.pptx --to png --scope /slide[3] -o slide-3.png
aioffice render report.docx --to pdf --engine auto -o report.pdf
aioffice audit report.docx
aioffice validate report.docx
```

![Visual QA loop: build, render, look, fix, rerender](assets/overview/visual-qa.svg)

Check the rendered output before shipping:

- No text is covered by another element.
- No label, footer, table, image, or chart spills outside its container.
- Headings and buttons fit their boxes.
- Chart labels, legends, axes, and data labels are readable.
- Screenshots show the actual final artifact, not a stale intermediate file.

For quick iteration, the Chromium renderer is fast. For closer Office fidelity,
use `--engine soffice` or `--engine auto` when LibreOffice and Poppler are
available. `aioffice doctor` reports renderer availability under `renderers`.

## Agent Workflow

A good agent does not guess. It reads the file, locates canonical paths, edits in
a batch, and verifies both structure and appearance:

```bash
aioffice read  report.docx --view outline
aioffice query report.docx "p[style=Heading1]"
aioffice get   report.docx /body/p[3]
aioffice edit  report.docx --ops @ops.json --expect-rev <rev>
aioffice validate report.docx
aioffice render report.docx --to png -o report.png
```

For visual documents, load the design vocabulary before editing:

```bash
aioffice help chart-polish
aioffice help themes
aioffice help masters
aioffice help properties-pptx
aioffice help properties-xlsx
aioffice help properties-docx
```

The design guide is intentionally strict: do not ship default-white slides,
bare charts, unstyled tables, or text boxes that have never been rendered and
looked at.

## Source Of Truth

The repository has several document layers:

- [CONTRACT.md](../CONTRACT.md) is the stable AI-facing contract.
- `aioffice schema` is the machine-readable live command surface.
- `aioffice help <topic>` is the runtime property and grammar reference.

- [CHANGELOG.md](../CHANGELOG.md) records additive package releases.
